# 2. Product Overview: Kuamini Threat Protection Agent

The Kuamini Threat Protection Agent is a production-ready, cross-platform endpoint security solution with a centralized management console and distributed desktop agents for Windows, macOS, and Linux. It is designed to provide enterprise-grade security with minimal resource footprint, making advanced protection accessible to SMBs.

## 3.7 Agent Architecture & Threat Detection

**Agent Stack:** Python 3.x, pystray, requests, psutil, Pillow, PyInstaller

**Key Agent Features:**
- System tray UI, real-time status, alerts
- Threat engine: file scanner, process monitor, network security
- Heartbeat (60s), full scan (1h), threat reporting
- HTTPS with CA bundle, HMAC-SHA256, encrypted config
- Audit logging, singleton enforcement, auto-start

**Threat Detection Engine:**
- Signatures, scanner, process monitor, engine, reporter
- Quick scan, full scan, realtime monitor (policy-controlled)
- Isolated from core agent (safe, non-blocking)

**Example Threat Detection Flow:**
1. Scan (quick/full/realtime) runs in daemon thread
2. Detects threat (signature, behavior, hash)
3. Generates report (metadata, severity)
4. POST /api/agent/threat
5. Console dashboard updates in real-time

**Performance:**
- Idle: <1% CPU, 30-50MB RAM
- Full scan: 20-40% CPU (during scan)
- Network: ~1-5KB per threat report

**Policy Control:**
```json
{
	"scheduled_scan": { "enabled": true, "interval": 3600, "mode": "full" },
	"realtime_monitor": { "enabled": false },
	"threat_reporting": { "enabled": true, "severity_threshold": 50 }
}
```

## 3.8 Security Architecture

- HTTPS/TLS, CA bundle, encrypted data in transit
- JWT token validation, HMAC-SHA256, rate limiting, input validation
- Row-level security, parameterized queries, audit logging
- Agent: token-based authentication, config encryption, singleton enforcement

## 3.9 Deployment & Operations

- AWS EC2 (frontend, API), AWS RDS (database), S3 (installers/backups)
- GitHub Actions CI/CD, PM2 process manager, nginx reverse proxy
- Environment variables for secrets, session, and DB config
- Automated build, test, lint, and deploy pipeline

## 3.10 Scalability & Performance

- Horizontal scaling: EC2 auto-scaling, nginx load balancing
- Database: Multi-AZ, read replicas, automated failover
- Agent: distributed, no agent-to-agent comms, scales with endpoints

## 3.11 Known Limitations & Roadmap

- Agent updates require manual install (auto-update planned)
- Single AWS region (multi-region planned)
- Limited threat intelligence feeds (expansion planned)
- Future: ML analytics, mobile app, agent clustering, advanced dashboard
